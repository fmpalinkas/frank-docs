## Purpose  

The Points of Interest (POI) Photos service provides photos of the POI.  

## Request data  

### HTTPS methods: `GET` `HEAD`  

For ease of viewing and identification:  

- Constants and parameters enclosed in curly brackets &#123; &#125; **must be replaced** with their values.
- Please see the following [Request parameters](#request-parameters) section with the required and optional parameters tables for their values. The generic request format is as follows.  

```shell [type=get] [title=Request URL format]
https://{baseURL}/maps/orbis/places/poiPhoto?key={Your_API_Key}&apiVersion={apiVersion}&id={id}&height={height}&width={width}
```

```shell [type=get] [title=Request curl command format]
curl 'https://{baseURL}/maps/orbis/places/poiPhoto?key={Your_API_Key}&apiVersion={apiVersion}&id={id}&height={height}&width={width}'
```

    <tr>
      <th scope="row">
        `width`  
        *integer*  
        `height`  
        *integer*
      </th>
      <td rowspan="2">
        The maximum height/width of the image.
        <ul>
          <li>
            If only one of them is provided, the image will be scaled according
            to that dimension.
          </li>
          <li>
            If none of them are provided, the service will return the original
            sized image.
          </li>
          <li>
            If both of them are provided, the service will fit, center, and crop
            the image to fill the desired dimensions.
          </li>
          <li>
            If the values provided are bigger than the original image
            dimensions, the service will return the original image.
          </li>
        </ul>
      </td>
    </tr>
  </tbody>
</table>

## Response data  

### Response body  

**Note:** The response body is a photo of the POI.

With respect to the Points of Interest Photos API, the following applies: 

<ul>

    Interest Details API and Points of Interest Photos API as being "powered by
    Foursquare".

    Interest Details API and Points of Interest Photos API to create, improve,
    or edit a venue or point of interest database.

### Error response  

The error response content type depends on the `ext` parameter.  

```json [title=Error response body with the request URL <ext> parameter value set to "json"]
{
  "message": "Missing parameter 'parameterName'",
  "detailedError": {
    "code": "MissingParameter",
    "message": "Missing required parameter 'parameterName'."
  },
  "httpStatusCode": "400"
}
```

```xml [title=Error response body with the request URL <ext> parameter value set to "xml"]
<ErrorEvResponse>
  <message>Missing parameter 'parameterName'</message>
  <detailedError>
    <code>MissingParameter</code>
    <message>Missing required parameter 'parameterName'.</message>
  </detailedError>
</ErrorEvResponse>
```
