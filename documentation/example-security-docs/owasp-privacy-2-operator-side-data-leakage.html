    OWASP Top 10 Privacy Risk Countermeasures

    OWASP is the acronym for the Open Web Application Security Project .

    2. Operator-side Data Leakage

    Risk Description

    Failure to prevent the leakage of any information containing or related to user data, or the data itself, or to any unauthorized party resulting in loss of data confidentiality.

    • Introduced either due to intentional malicious breach or unintentional mistake.

    • This can be caused by insufficient access management controls, insecure storage, duplication of data, or a lack of awareness.

    How to check?

    1. Research the reputation and reliability ofthe operator

      1. Have there been former breaches related to the operator?

      2. Does the provider proactively prove privacy and security and if yes, how?

      3. Is there a bug bounty program to report vulnerabilities?

      4. Is the provider certified according to ISO 27001 or ISO 27018 (cloud providers)?

      5. Is the operator located in a country with high privacy standards?

    2. Audit the operator

      1. Are privacy best practices in place?

      2. Is awareness training mandatory for all employees?

      3. Is there a privacy engineering team?

      4. How is personal data anonymized?

      5. Is personal data encrypted?

      6. Who has access to the data (need-to-know principle)?

    3. Audit methods

      1. Paperbased audit (fair)

      2. Interviewbased audit (good)

      3. Onsite audit and systemchecks

        (best)
    Countermeasures

    1. Appropriate Identity and Access Management (physical as well as logical):

      1. Principle of least privilege.

    2. Use strong encryption for all personal data stored (data at rest) especially on mobile media:

      1. USB memory sticks

      2. Laptophard disks

      3. Tablet and phone local storage

      4. Backup tapes

      5. Portable hard disk drives

    3. Awareness training for all employees regarding handling of personal data.

    4. Implementation of a data classification and information handling policy.

    5. Monitor and detect classified data when it leaks from endpoints, web portals and cloud services (e.g. by Data Leakage Prevention, SIEM).

    6. Implement Privacy by Design

    7. Anonymisation of personal data:

      1. It is common practice to anonymise personal data and use it for other purposes e.g., testing or marketing.

      2. Anonymisation is not easy (e.g., AOL search data leak ) and there are many anonymisation theories which can be very complex.

      3. Pseudonymisation which means that data can only be connected to a person with the help of a third party that knows the person and corresponding pseudonym.
