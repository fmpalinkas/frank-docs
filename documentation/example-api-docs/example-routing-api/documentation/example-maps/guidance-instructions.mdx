## Purpose

This document describes route planning parameters that are related to the guidance.
They can be used, among others, in the [Calculate Route](/routing-api/documentation/Example-maps/calculate-route) service.

Please be aware that this document does not specify the complete request or response format. 
To maintain forward compatibility you must be able to process responses with additional non-documented data fields. 
Our deprecation policy only covers the documented subset of these formats.

## Types

## Request data

### HTTPS method: `GET` &nbsp; `POST`

Constants and parameters enclosed in curly brackets &#123; &#125; **must be replaced** with their values.

#### Generic request format

The following request URL contains all of this endpoint&#39;s respective parameters. For their details, please see the <a href="/routing-api/documentation/Example-maps/common-routing-parameters" target="_blank">Common Routing Parameters</a> page.

```shell [type=get] [title=Generic request example]
https://{baseURL}/routing/{versionNumber}/calculateRoute/{routePlanningLocations}/{contentType}?key={Your_API_Key}
&instructionsType={instructionsType}
&language={language}
&sectionType={sectionType}
&instructionAnnouncementPoints={string}
&instructionPhonetics={string}
&instructionRoadShieldReferences={string}
```

#### GET URL example  

**Note:** Linebreaks are designated by "&bsol;".

```shell [type=get] [title=Request URL example]
https://api.Example.com/routing/1/calculateRoute/\
52.50931,13.42936:52.50274,13.43872/json?\
&instructionsType=text&language=en-GB\
&sectionType=lanes&instructionAnnouncementPoints=all\
&instructionPhonetics=LHP&instructionRoadShieldReferences=all\
&key={Your_API_Key}
```

#### GET curl command example  

**Note:** Linebreaks are designated by "&bsol;".

```shell [type=get] [title=Request curl command example]
curl -X GET "https://api.Example.com/routing/1/calculateRoute/\
52.50931,13.42936:52.50274,13.43872/json?\

&sectionType=lanes&instructionAnnouncementPoints=all\

```

      <th scope="row">
        <code>sectionType</code>  
        *string*
      </th>
      <td>
        The following section types can additionally be requested:
        <ul>
          <li>
            <code>lanes</code>: Sections with lane information. This is a
            combination of lane directions (<code>directions</code>), including
            the follow direction (<code>follow</code>) where recommended, and
            lane markings on the road (<code>laneSeparators</code>).
            <ul>
              <li>
                Lanes are given left-to-right with the direction of travel.
              </li>
              <li>
                Separators are also given left-to-right, starting with the
                marking to left of the left-most lane and ending with the
                marking to the right of the right-most lane, such that
                <code>laneSeparator[x]</code> is to the left of <code>
                  lane[x]
                </code>.
              </li>
            </ul>
          </li>
        </ul>
      </td>
    </tr>
  </tbody>
</table>

        <code>instructionsType</code>  
        *string*
      </th>
      <td>
        If specified, guidance instructions will be returned (if available).
        Possible values are:
        <ul>
          <li>
            <code>coded</code>: returns raw instruction data without
            human-readable messages.
          </li>
          <li>
            <code>text</code>: returns raw instructions data with human-readable
            messages in plain text.
          </li>
          <li>
            <code>tagged</code>: returns raw instruction data with tagged
            human-readable messages to permit formatting.
          </li>
        </ul>
        <dl>
          <dt>Note:</dt>
          <dd>
            The <code>instructionsType</code> parameter cannot be used in
            conjunction with <code>routeRepresentation=none</code>.
          </dd>
          <dd>
            If alternative routes are requested, instructions will be generated
            for each route returned.
          </dd>
          <dd>
            See the <a href="#tagged-messages">Tagged messages</a> section for
            more information about tagged messages.
          </dd>
        </dl>
        <b>Values:</b>
        <ul>
          <li>
            <code>coded</code>
          </li>
          <li>
            <code>text</code>
          </li>
          <li>
            <code>tagged</code>
          </li>
        </ul>
      </td>
    </tr>
    <tr>
      <th scope="row">
        <code>language</code>  
        *string*
      </th>
      <td>
        The <code>language</code> parameter determines the language of the
        returned names and messages.
        <ul>
          <li>
            Proper nouns (the names of streets, plazas, etc.) are returned in
            the specified language, or if that is not available, they are
            returned in an available language that is close to it.
          </li>
          <li>
            Allowed values are (a subset of) the IETF language tags described
            here.
          </li>
          <li>
            The currently supported languages are listed in the
            <a href="#supported-languages">Supported languages</a> section.
          </li>
        </ul>
        <b>Default value:</b> <code>en-GB</code>
      </td>
    </tr>
    <tr>
      <th scope="row">
        <code>instructionAnnouncementPoints</code>  
        *string*
      </th>
      <td>
        <p>
          Specifies whether to include announcement points in instructions. If
          specified, the instruction will include up to three additional
          fine-grained announcement points, each with their own location,
          maneuver type, and distance to the instruction point.
        </p>
        <p>Possible values are:</p>
        <ul>
          <li>
            <code>none</code>: Do not include instruction announcement points in
            instructions.
          </li>
          <li>
            <code>all</code>: Include instruction announcement points in
            instructions, exposing the following instruction fields, where
            applicable:
            <ul>
              <li>
                <code>earlyWarningAnnouncement</code>
              </li>
              <li>
                <code>mainAnnouncement</code>
              </li>
              <li>
                <code>confirmationAnnouncement</code>
              </li>
            </ul>
            Each of these announcements has the following fields:
            <ul>
              <li>
                <code>point</code>: Location of the announcement. It is a
                <code>point</code> object (see <a href="#types">Types</a>).
              </li>
              <li>
                <code>pointIndex</code>: Largest index in the polyline of the
                route for which
                <code>points[pointIndex]</code> lies on or before <code>
                  point
                </code>.
              </li>
              <li>
                <code>maneuver</code>: Maneuver code. Possible values include all possible values of the <a href="/routing-api/documentation/Example-maps/calculate-route#maneuver-codes" target="_blank"><code>maneuver</code></a> instruction field as well as all <a href="#extra-maneuver-codes-for-instruction-announcement-points">extra maneuver codes</a> for instruction announcement points.
              </li>
              <li>
                <code>distanceInMeters</code>: Distance in meters from
                <code>point</code>
                to the <code>point</code> in <code>instruction</code>.
              </li>
            </ul>
            The semantics of <code>point</code>, <code>pointIndex</code>, and
            <code>maneuver</code> are equivalent to their counterparts of the same
            name in
            <code>instruction</code>.
          </li>
        </ul>
        <p>
          <b>Default value:</b> <code>none</code>.
        </p>
      </td>
    </tr>

### Structure of a successful response

<table style={{ overflowWrap: 'anywhere' }}>
  <thead>
    <tr>
      <th scope="col" style={{ fontSize: '1em', width: '300px' }}>

  <tbody>
    <tr>
      <th scope="row">
        <code>guidance</code>  
        *object*
      </th>
      <td>
        This contains guidance related fields. This field is present only when
        guidance was requested and is available.
      </td>
    </tr>
    <tr>
      <th scope="row">
        <code>instruction</code>  
        *object*
      </th>
      <td>
        A set of attributes describing a maneuver, e.g., "Turn right", "Keep
        left", "Take the ferry", "Take the motorway", "Arrive".
      </td>
    </tr>
    <tr>
      <th scope="row">
        <code>instructionGroup</code>  
        *object*
      </th>
      <td>
        This groups a sequence of instruction elements which are related to each
        other.
        <ul>
          <li>
            The sequence range is constrained with
            <code>firstInstructionIndex</code> and
            <code>lastInstructionIndex</code>.
          </li>
          <li>
            When human-readable text messages are requested for guidance (
            <code>instructionType=text</code> or <code>tagged</code>), then the
            <code>instructionGroup</code> has a summary message returned when
            available.
          </li>
        </ul>
      </td>
    </tr>
    <tr>
      <th scope="row">
        <code>roadShieldReferences</code>  
        <code>signpostRoadShieldReferences</code>  
        *arrays of <code>roadShieldReference</code> objects*
      </th>
      <td>
        A <code>roadShieldReference</code> object describes a single road shield
        reference. Please refer to the <a href="/routing-api/documentation/Example-maps/calculate-route#structure-of-a-road-shield-reference-object" target="_blank">
        structure of a road shield reference object</a> 
        for further information.
      </td>
    </tr>
  </tbody>

### Structure of the `instruction` object

<Blockquote type="announcement" hasIcon>
  #### Guidance instruction deprecation policy
  <p>
    As part of the continuous improvement of the Routing API, we also extend our
    guidance instructions with new features.
  </p>
  <p>
    In many cases, this requires the introduction of new maneuver codes for new
    types of instructions. Hence we cannot guarantee the same deprecation policy
    for guidance instructions as compared to the rest of the Routing API.
  </p>
  <p>
    In particular, clients are expected to be forward compatible, and we reserve
    the right to extend all enums defined below with new values. This includes
    completely new maneuver codes for new types of instructions.
  </p>
  <p>
    At the same time we reserve the right to retire enum values for instructions
    that are superseded by their improved counterparts no longer generated by the
    internal Guidance component. The same rules for
    forward-/backward-compatibility apply to fields for guidance instructions.
  </p>
  <p>
    For the retirement of instructions that are no longer relevant, we give a
    deprecation period of 4 months. After this period we reserve the right to
    remove deprecated values at our convenience.
  </p>
</Blockquote>

### Example of a combined message

Sometimes it is possible to combine two successive instructions into a single instruction making it easier to follow. When this is the case the `possibleCombineWithNext` flag will be `true`. For example:

```shell [title=possibleCombineWithNext flag set to true]
10. Turn left onto Einsteinweg/A10/E22 towards Ring Amsterdam
11. Follow Einsteinweg/A10/E22 towards Ring Amsterdam
```

The `possibleCombineWithNext` flag on instruction 10 is `true`. This indicates to the clients of coded guidance that it can be combined with instruction 11. The instructions will be combined automatically for clients requesting human-readable guidance. The `combinedMessage` field contains the combined message:

```shell [title=combinedMessage field]
Turn left onto Einsteinweg/A10/E22 towards Ring Amsterdam then follow Einsteinweg/A10/E22 towards Ring Amsterdam.
```
