    OWASP Top 10 Privacy Risk Countermeasures

    OWASP is the acronym for the Open Web Application Security Project .

    5. Nontransparent Policies, Terms, and Conditions

    Risk Description

    Not providing sufficient information to describe how data is processed, such as its collection, storage, processing and deletion.

    • Failure to make this information easily accessible and understandable for non-lawyers.

    How to check?

    Check if policies, terms and conditions:

    1. Are easy to find.

    2. Fully describe data processing:

      1. Who are you / who is processing the data?

      2. Including data transfers.

      3. Analysis performed.

      4. Retention time.

      5. Meta data used.

      6. What are the rights?

    3. Understandable for non-lawyers.

    4. Complete, but KISS (Keep it short and simple).

    5. Include a process for obtaining user consent if the terms, policies or conditions change.

    6. Are available in the user’s language

    7. Explain which data are collected.

    8. Explain the purposes for which personal data is collected.

    9. Use a readability tester like Readable.io to check whether a text is hard to read or not.

    10. Are privacy rules actively communicated or does the user have to take action?

    Countermeasures

    1. Terms & Conditions (T&Cs) should be specifically for the use and data processing of the website.

    2. They should be easy to understand for non-lawyers and not too long.

    3. Provide an easily readable summary of the terms and conditions as well as a long version.

    4. Pictograms can be used for visual aids.

    5. Use separate T&Cs for use and data processing.

    6. Use release notes to identify change history of T&Cs, and policies/notices over time.

    7. Keep track of which users consented to which version and any other time at which they may opt in to newer versions.

    8. Deploy "Do Not Track" on the server side.

    9. When collecting information it should be clear why it is needed.

      1. You should also try to predict whether you will be likely to do other things with it in the future and tell the users if you have such plans.

    10. Provide a list of cookies, widgets, etc., used with an explanation of their use e.g., sharing data or advertising.

    11. Provide an opt-out cookie button for the users.
