## Purpose  

The Copyrights API is designed to serve copyright information for the Map Display services. As an alternative to copyrights for map request, you can receive copyrights for the map service called **captions**.

### Run this endpoint

You can easily run this and other endpoints. Go to the Example  [API Explorer](/map-display-api/api-explorer/)  page and follow the directions.

## Map Copyrights

### Request data

### HTTPS method: `GET`

Please see the following [Request parameters](#request-parameters-map-copyrights) section with the required parameters table for these values. The generic URL format is as follows.

<h4 style={{ fontSize: '1em' }}>URL format</h4>

The schema for the map copyrights request:

```shell [type=get] [title=Request URL]
https://{baseURL}/map/{versionNumber}/copyrights?key={Your_API_key}
```

<h4 style={{ fontSize: '1em' }}>Example</h4>

```shell [type=get] [title=Request example]
https://api.Example.com/map/2/copyrights?key={Your_API_Key}
```

<h4 style={{ fontSize: '1em' }}>curl command format</h4>

```shell [type=get] [title=Request curl command]
curl 'https://api.Example.com/map/2/copyrights?key={Your_API_Key}'
```

based on https://www.Example.com/en_gb/thirdpartyproductterms/

### Response data

For the map service copyright request, the response body will contain a copyrights caption in JSON format.

```json [title=JSON response example]
{
  "formatVersion": "0.0.1",
  "copyrightsCaption": "©Example"
}
```

### Error response

The Map Display API Copyrights service for an invalid request returns a response body in XML or JSON format. The XML format is returned by default. To have an error response returned in JSON format, `application/json` has to be specified in the  **Accept**  HTTP request header.
