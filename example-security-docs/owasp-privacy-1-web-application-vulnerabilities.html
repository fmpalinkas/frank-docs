    OWASP Top 10 Privacy Risk Countermeasures

    OWASP is the acronym for the Open Web Application Security Project .

    1. Web Application Vulnerabilities

    Risk Description

    Vulnerability is a key problem in any system that guards or operates on sensitive user data.

    • Failure to suitably design and implement an application, detect a problem or promptly apply a fix (patch) is likely to result in a privacy breach.

    • This risk also encompasses the OWASP Top 10 List of web application vulnerabilities and the risks resulting from them.

    How to check?

    1. Are regular penetration tests performed with a focus on privacy?

    2. Are developers trained regarding web application security?

    3. Are secure coding guidelines applied?

    4. Is any of the used software out of date (server, database, frameworks, other infrastructure components)?

    Countermeasures

    1. Perform regular penetration tests by independent security experts.

    2. Track remediation of findings.

    3. Train application developers and architects in secure development.

    4. Apply procedures for secure development (e.g. Security Development Lifecycle SDL).

    5. Install updates, patches and hotfixes on a regular basis.

    Examples

    1. Injection Flaws allow attackers among others to copy or manipulate data by attacks like SQL injection.

    2. Sensitive Data Exposure allows attackers gather sensitive information e.g., due to missing encryption with a man-in-the-middle attack.

    3. Use of Insecure Direct Object References allows attackers to guess and access sensitive information, especially if access control is missing.

    4. Use of Components with Known Vulnerabilities, e.g., unpatched software flaws, and Security Misconfigurations , e.g. unhardened application platform.

    5. In general it is possible for attackers to gain access to, manipulate or delete personal data that the application is processing by abusing rights, entering malicious code or eavesdropping on communications.
