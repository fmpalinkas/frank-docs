## Overview  

This Example Release Notes page records the release date with changes and bug fixes associated with a product release. The release dates are in descending order starting with the most recent update. 

### Release date: 2024.06.28
**Changes:**
- [Geocode](../geocode):
- [Structured Geocode](../structured-geocode):
  - Added a new field [`building`](../geocode#building) in the response of type Point Address (PAD) for respective countries - for example Saudi Arabia.

### Release date: 2024.06.25
**Changes:**
- [Geocode](../geocode)
  - Example New Maps Geocode API: Migration guide added.

### Release date: 2022.10.28
**Changes:**  

- Added the Example Geocoding API as a stand-alone API with the following endpoints:
  -  `GET`   [Geocode](../geocode)
  -  `GET`   [Structured Geocode](../structured-geocode)

### Release date: 2022.05.09

**Changes:**  

- All endpoints:  
  - Added the [Region-specific content](/search-api/documentation/product-information/region-specific-content) document for South Korea.
  - Updated the [Supported Language](/search-api/documentation/product-information/supported-languages) reference list to include Korean Latin and Hangul scripts.

### Release date: 2022.04.14

**Changes:**    

- [Geocode](../geocode):
  - Added a new field `matchConfidence` in the response for all result types.

### Release date: 2021.03.05  

**Changes:**    

- [Geocode](../geocode):
  - Updated the response object for geographies of type `PostalCodeArea` in the USA with a new field `postalName`, which represents the name for a postal code that is related to a single administrative area, city, town, or other populated place.
  - Updated the field `freeFormAddress` of `PostalCodeArea` in the USA which will now contain `postalName`.
- [Structured Geocode](../structured-geocode):
  - Updated the response object for geographies of type `PostalCodeArea` in the USA with a new field `postalName`, which represents the name for a postal code that is related to a single administrative area, city, town, or other populated place.
  - Updated the field `freeFormAddress` of `PostalCodeArea` in the USA which will contain `postalName`.

### Release date: 2021.02.16  

**Changes:**   

- [Geocode](../geocode):
  - Updated the response `address` object which has a new `countrySubdivisionCode` field in line with ISO 3166-2.
- [Structured Geocode](../structured-geocode):
  - Updated the response `address` object which has a new `countrySubdivisionCode` field in line with ISO 3166-2. 

### Release date: 2021.01.07  

**Changes:**  

- All endpoints: 
  - Updated: In USA, the `LocalName` is now aligned with postal city/town names.

### Release date: 2020.12.08

**Changes:**  

- All endpoints: 
  - Added a new  `entityTypeSet`  parameter that can be used to restrict the result to a specific entity type.
