## Purpose

If your search use case only requires  `POI` (Points of Interest) results filtered by category, you may use the Category Search endpoint. This endpoint will only return  `POI`  results which are categorized as specified.

### Run this endpoint

You can easily run this and other endpoints. Go to the Example  [API Explorer](/search-api/api-explorer/)  page and follow the directions.

## Request data

### HTTPS Method: `GET`

For ease of viewing and identification:

- Constants and parameters enclosed in curly brackets &#123; &#125; **must be replaced** with their values.
- Please see the following [Request parameters](#request-parameters) section with the required and optional parameters tables for their values. The generic request format is as follows.

```shell [type=get] [title=URL request format]
https://{baseURL}/search/{versionNumber}/categorySearch/{query}.{ext}?key={Your_API_Key}&typeahead={typeahead}&limit={limit}&ofs={ofs}&lat={lat}&lon={lon}&countrySet={countrySet}&radius={radius}&topLeft={topLeft}&btmRight={btmRight}&language={language}&extendedPostalCodesFor={extendedPostalCodesFor}&categorySet={categorySet}&brandSet={brandSet}&connectorSet={connectorSet}&fuelSet={fuelSet}&vehicleTypeSet={vehicleTypeSet}&view={view}&openingHours={openingHours}&timeZone={timeZone}&mapcodes={mapcodes}&relatedPois={relatedPois}&minPowerKW={minPowerKW}&maxPowerKW={maxPowerKW}
```

```shell [type=get] [title=URL request example]
https://api.Example.com/search/2/categorySearch/pizza.json?key={Your_API_Key}
```

```shell [type=get] [title=curl command request example]
curl 'https://api.Example.com/search/2/categorySearch/pizza.json?key={Your_API_Key}'
```

      <th>
         `type`<br />

      </th>
      <td>
        Type of mapcode.<br />
         `Local`<br />
        The shortest possible (and easiest to remember) mapcode. Local mapcodes
        are especially useful when the user knows what territory the mapcode is
        in (for example, when an application is designed to be used inside just
        one European country or US state). Note that the code element of a Local
        mapcode is ambiguous when used without the territory element. For
        example, the &quot;4J.P2&quot; mapcode can mean the Eiffel Tower
        location ( `48.858380, 2.294440` ) (with the territory set to
         `FRA` ), but also some place in Amsterdam-Noord, Netherlands
        (52.382184, 4.911021) (with the territory set to  `NLD` ).<br />
         `International`<br />
        This mapcode is unambiguous. It is also the longest.<br />
         `Alternative`<br />
        Alternatives to Local mapcodes. Each Alternative mapcode points to
        slightly different coordinates due to the way mapcodes are computed (see
        the
         [mapcode documentation](http://www.mapcode.com/documentation)
        ). For example, the position from a response can be encoded as
        &quot;5DM.WC&quot; ( `51.759244, 19.448316` ) and the
        &quot;VHJ.036&quot; ( `51.759245, 19.448264` ), which are close
        to each other, but not exactly the same place.
      </td>
    </tr>
    <tr>
      <th>
         `fullMapcode`  <br />
         *string*
      </th>
      <td>
        The full form of a mapcode ( `territory`  +  `code` ).
        It is always unambiguous. The  `territory`  element is always
        in the Latin alphabet. In an  `International`  mapcode, the
         `territory`  part is empty.
      </td>
    </tr>
    <tr>
      <th id="territory">
         `territory`  <br />
         *string*
      </th>
      <td>
        The  `territory`  element of the mapcode.
        <ul>
          <li>
            The  `territory`  element is always in the Latin
            alphabet.
          </li>
          <li>
            Usually, the  `territory`  is the
             [ISO 3166-1 alpha 3](https://en.wikipedia.org/wiki/ISO_3166-1_alpha-3)
            abbreviation for the country name.
          </li>
          <li>
            However in these eight very large countries:
            <ul>
              <li>USA</li>
              <li>Canada</li>
              <li>Mexico</li>
              <li>Brazil</li>
              <li>India</li>
              <li>Russia</li>
              <li>China</li>
            </ul>
            an address has little meaning unless the user also knows what state
            it&#39;s in (just as, elsewhere, an address has little meaning if
            the user doesn’t know what country it’s in). More information about
             `territory`  codes is available at
             [Territories for mapcodes](http://www.mapcode.com/isos.html). This field is not returned for an  `International`
            mapcode.
          </li>
        </ul>
      </td>
    </tr>
    <tr>
      <th id="code">
         `code`<br />
         *string*
      </th>
      <td>
        The mapcode  *without*  the  `territory`  element. It

         `code`  is using the same language and alphabet as the
        response. The  `language`  parameter may be used to modify the
        language and alphabet of both the response and the  `code`

        Use  `fullMapcode`  instead.

### Error response

The error response content type depends on the `ext` parameter.

```json [title=Error response example - JSON]
{
  "errorText": "Error parsing 'language': Language tag 'en-ES' not supported",
  "detailedError": {
    "code": "BadRequest",
    "message": "Error parsing 'language': Language tag 'en-ES' not supported",
    "target": "language"
  },
  "httpStatusCode": "400"
}
```

```xml [title=Error response example - XML]
<?xml version="1.0" encoding="UTF-8"?>
<response>
  <errorText>Error parsing 'language': Language tag 'en-ES' not supported</errorText>
  <detailedError>
    <code>BadRequest</code>
    <message>Error parsing 'language': Language tag 'en-ES' not supported</message>
    <target>language</target>
  </detailedError>
  <httpStatusCode>400</httpStatusCode>
</response>
```
