## Purpose  

This document provides a description of internal parameters and response fields.  
  
## Request data  

### HTTPS Method: `GET`  

For ease of viewing and identification:  

- Constants and parameters enclosed in curly brackets &#123; &#125; **must be replaced** with their values.
- Please see the following [Internal request parameters](#internal-request-parameters) section with the optional parameters table for their values. The generic request format is as follows.  

**Important note:** Either `apiVersion` parameter or `Example-Api-Version` header needs to be present.  

```shell [type=get] [title=URL request format]
https://{baseURL}/maps/New/places/search/{query}.{ext}?key={Your_API_Key}&apiVersion={apiVersion}&paymentOptions={paymentOptions}
```

```shell [type=get] [title=URL request example]
https://api.Example.com/maps/New/places/search/pizza.json?key={Your_API_Key}&apiVersion=1&paymentOptions=true
```

```shell [type=get] [title=curl command request example]
curl 'https://api.Example.com/maps/New/places/search/pizza.json?key={Your_API_Key}&apiVersion=1&paymentOptions=true'
```

### Internal request parameters  

The following table describes additional internal parameters that can be used in a request.  

- If there is a default value that will be assumed when an optional parameter is not used, it is shown in the table.
- The order of request parameters is not important.  

<table data-notbreakword>
  <thead>
    <tr>
      <th scope="col" style={{ fontSize: '1em' }}>Optional&nbsp;parameters</th>
      <th scope="col" style={{ fontSize: '1em' }}>Description</th>
    </tr>
  </thead>
    <tbody>
      <tr>
        <th>
          `apiVersion`  
          *string*
        </th>
        <td>
          Service version. If the `Example-Api-Version` header is not present 
          the `apiVersion` parameter is required.  
          **Value:** A service version value. The current value is `1`.  
        </td>
    </tr>
    <tr>
      <th>
        `paymentOptions`  
        *boolean*
      </th>
      <td>
        If the `paymentOptions` flag is set to `true` then
        the [`paymentOptions`](#paymentoptions-array)
        array will be returned if it exists.  
        **Default value:** `false`
      </td>
    </tr>
    <tr>
      <th>
        `hydrogenPressure`  
        *boolean*
      </th>
      <td>
        Hydrogen Pressure represents the pressure a Petrol Station is capable of
        delivering hydrogen at. If the `hydrogenPressure` flag is
        set, the supported hydrogen pressure types are returned. This is only
        relevant for fuel station POIs that provide hydrogen as one of their
        available fuel types.  
        **Default value:** `false`  
      </td>
    </tr>
  </tbody>
</table>
