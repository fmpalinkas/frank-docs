    OWASP Top 10 Privacy Risk Countermeasures

    OWASP is the acronym for the Open Web Application Security Project .

    8. Outdated Personal Data

    Risk Description

    The use of outdated, incorrect or bogus user data. Failure to update or correct the data.

    How to check?

    1. Ask the operator how it is ensured that personal data is up-to-date.

    2. Check for possibilities to update personal data in the application.

    3. Are there regular checks to validate that data is up-to-date (e.g., "please verify your shipping address")?

    4. Question how long it is likely that data is up-to-date, and how often it usually changes.

    Countermeasures

    1. Implement a procedure to update the user's personal data by obtaining inputs from them after a certain time period.

    2. The user should approve data if he or she is triggering a "critical" action.

    3. Provide a form to enable users to update their data.

    4. In case of an update, make sure to forward the information to any third parties / subsystems that previously received the user's data (if there are any).

    Examples

    • An update form is provided on the website so that the user can update his or her data when needed.

    • Amazon is asking whether your address and account data is correct before you can finish your order (CRM clearing).
