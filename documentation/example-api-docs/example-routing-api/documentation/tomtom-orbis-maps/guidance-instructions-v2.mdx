## Purpose

This document describes route planning parameters that are related to guidance instructions.  
They can be used in the [Calculate Route](./calculate-route) service.

Please be aware that this document does not specify the complete request or response format.
To maintain forward compatibility you must be able to process responses with additional non-documented data fields.
Our deprecation policy only covers the documented subset of these formats.

## Request data

### HTTPS methods: `GET`&nbsp;`POST`

- Constants and parameters enclosed in curly brackets &#123; &#125; **must be replaced** with their values.

#### Generic request format

The functionality described here extends the functionality of the endpoint described in <a href="./calculate-route" target="_blank">Calculate Route</a>.

**Note:** Only parameters that behave differently from the ones listed in <a href="./calculate-route" target="_blank">Calculate Route</a> are explicitly listed.

```shell [type=get] [title=URL request example]
https://{baseURL}/maps/New/routing/calculateRoute/{routePlanningLocations}/json?key={Your_API_Key}
&instructionsType=coded
&guidanceVersion=2
&instructionPhonetics={instructionPhonetics}
&instructionRoadShieldReferences=all
&roadAndExitIconReferencesVersion={string}
&language={language}
&sectionType={sectionType}
```

### Extensions of existing features

#### instructionsType parameter

To enable Guidance Instructions, you need to set the <code>instructionsType</code> parameter.
The only allowed value is <code>coded</code>.  
If set, <code>guidanceVersion</code>, <code>instructionPhonetics</code>, <code>instructionRoadShieldReferences</code> and <code>language</code> also need to be set.
